# 주식 자동 매매 프로그램 (Stock Trading GPT) 로드맵 및 아키텍처 리뷰

현재 프로젝트는 **"데이터 수집 ➡️ 기술적 분석 ➡️ 대시보드 시각화"**까지 훌륭하게 구현되어 있습니다. 하지만 최종 목표인 **'자동 매매(Algorithmic Trading)'**를 달성하기 위해서는 시스템이 사람의 개입 없이 스스로 판단하고 매매를 실행할 수 있는 구조가 추가되어야 합니다.

앞으로 추가해야 할 핵심 기능들과 고도화 포인트를 5가지 영역으로 나누어 제안합니다.

---

## 1. 🚨 필수: 매매 실행 및 계좌 관리 (Execution & Portfolio)

현재는 분석(관망, 매수 검토 등)만 하고 실제 주문을 넣는 로직이 없습니다. 다음 API 연동이 추가되어야 합니다.

*   **매수/매도 주문 API 연동**: 한국투자증권 해외주식 주문 API(정정/취소 포함) 연동 로직 (`kis_client.py` 확장)
*   **잔고 및 예수금 조회 API**: 
    *   현재 계좌에 달러(USD) 예수금이 얼마나 있는지 파악하여 주문 가능 수량을 계산해야 합니다.
    *   이미 보유 중인 주식의 매입 단가와 수익률을 파악해야 합니다.
*   **체결 확인 로직**: 주문을 넣었다고 100% 체결되는 것이 아니므로, 체결 내역을 조회하고 미체결 시 호가를 정정하는 로직이 필요합니다.

## 2. ⏰ 필수: 스케줄러 및 백그라운드 자동화 (Automation)

현재 구조는 사용자가 웹 브라우저 접속(FastAPI 엔드포인트 호출)을 해야만 코드가 실행됩니다. 진정한 '자동 매매'를 위해서는 백그라운드에서 주기적으로 돌아가는 스케줄러가 필요합니다.

*   **스케줄러 도입 (APScheduler, Celery 등)**:
    *   **장 시작 전**: 환율 확인, 계좌 잔고 확인, 당일 매매 대상(스크리닝) 1차 추출
    *   **장 중 (미국장 23:30 ~ 06:00)**: N분 단위로 가격을 모니터링하며 매수/매도 조건 도달 시 자동 주문 실행
    *   **장 마감 후**: 일일 성과(PnL) 결산 및 DB 저장
*   **장 운영시간 관리**: 미국 휴장일, 썸머타임 적용 여부 등을 체크하여 스케줄러가 동작하도록 제어해야 합니다.

## 3. 🛡️ 고도화: 리스크 관리 및 포지션 사이징 (Risk Management)

단순히 지표가 좋다고 전액 매수하면 큰 손실을 볼 수 있습니다.

*   **포지션 사이징 (투자 비중 조절)**: 1개 종목에 총 자산의 최대 몇 %까지만 투자할 것인지 규칙 설정 (예: 켈리 베팅, 고정 비율법)
*   **손절매(Stop Loss) / 익절(Take Profit) 로직**:
    *   보유 종목이 매입가 대비 -5% 하락 시 기계적 손절매 주문
    *   목표가(+10%) 도달 시 분할 매도 또는 트레일링 스탑(Trailing Stop, 수익 보존) 적용

## 4. 🧠 고도화: "Trading GPT"에 걸맞는 AI 융합 (LLM 연동)

프로젝트 이름이 `Stock Trading GPT`인 만큼, 단순히 전통적인 기술적 지표(RSI, MACD)만 보는 것을 넘어 LLM(ChatGPT, Gemini 등)의 판단을 결합하면 훨씬 강력해집니다.

*   **종합 판단 프롬프트 구성**:
    *   기술적 지표 값 (현재 코드에서 산출된 `analysis` 데이터)
    *   한국투자증권 "해외뉴스종합" API 등을 통해 긁어온 해당 종목의 최신 뉴스 기사 요약
    *   위 데이터를 GPT-4나 Gemini API로 보내어 **"현재 시장 상황과 뉴스를 고려했을 때, 이 종목을 매수하는 것이 합리적인지 분석하고 1~10점으로 평가해줘"**라는 식의 AI 시그널을 최종 매매 로직에 반영합니다.

## 5. 📊 고도화: 데이터베이스 및 알림 시스템 (Logging & Notification)

*   **데이터베이스 (SQLite, PostgreSQL 등)**:
    *   주문 내역, 체결 단가, 일일 수익률 현황을 DB에 저장해야 나중에 전략의 문제점을 백테스트하고 복기할 수 있습니다.
*   **메신저 알림 (텔레그램, 슬랙 등)**:
    *   서버에 항상 접속해 있을 수 없으므로, 봇이 매수/매도를 체결했을 때나 서버에 오류가 났을 때 즉시 스마트폰 텔레그램으로 알림을 보내주는 기능이 필수입니다.

---

### 🚀 향후 개발 추천 순서
1. `kis_client.py`에 **잔고 조회**와 **주문(매수/매도)** 함수 추가하기
2. FastAPI 앱 내에 **스케줄러**를 붙여서 정해진 시간에 조건 검색 -> 분석 -> 매수 프로세스가 자동으로 흘러가게 만들기
3. 거래 발생 시 텔레그램으로 메시지 쏘기
4. 매수한 종목을 모니터링하다가 수익률 기반으로 익절/손절 매도 로직 붙이기
5. LLM API 연동하여 최종 의사결정에 반영하기
